HUGO from docker

August 6, 2025

This post is about my short jurney with HUGO, static site generator.

My expectations was:

  • As usual, I prefer docker - if possible.
  • I want to backup and track my work on Github.
  • And I expect speed and simplicity - of course.

So I started with creating parent folder HUGO. I this folder I created new site sbezo.github.io with HUGO container and initialized git.

mkdir HUGO
cd HUGO
docker run --rm -v $(pwd):/src hugomods/hugo:debian-nightly new site sbezo.github.io --format yaml
git init

-- format yaml is used, becase I choose PaperMod theme. Each theme has its own configuration preferencies. Without this switch default ‘.toml’ format of config file will be used.

I prepared docker-compose.yml file directly in HUGO folder for easy spinning up of my site locally.

services:
  server: 
      image: hugomods/hugo:debian-nightly
      command: server -D --poll 700ms
      volumes:
        - "./sbezo.github.io:/src"
      ports:
        - "1313:1313"

Now I have a new site as a subfolder sbezo.github.io with default setting.
As I choose PaperMod theme, next step was to install it to my newly generated site. So:

cd sbezo.github.io
git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

And then I have to configure choosen theme - just replaced default hugo.yaml with following setup.
Again, the most simple as possible;)

baseURL: "https://sbezo.github.io"
languageCode: en-us
title: sbezo.github.io

theme: PaperMod

params:
  profileMode:
    enabled: true
    title: "Stefan Bezo"
    subtitle: "Sometimes, I just feel like putting thoughts into words ;)"
    imageUrl: "/img/me.jpeg"
    imageWidth: 120 # custom size
    imageHeight: 120 # custom size
    
  socialIcons: # optional
    - name: "linkedin"
      url: "https://www.linkedin.com/in/stefanbezo"
    - name: "github"
      url: "https://github.com/sbezo"

  defaultTheme: auto
  #ShowShareButtons: true
  ShowCodeCopyButtons: true
  ShowPostNavLinks: true


menus:
  main:
  - name: Blog
    pageRef: /posts
    weight: 10

I copied file me.jpeg to folder assets/img

Now its time to bring up my site locally. Simple as that:

docker compose up

And Finaly - add my first post.
Check name of running comtainer, attach shell to it and create new post with HUGO:

docker ps
docker exec -it hugo_2025-server-1 bash
hugo new posts/HUGO.md

Now I can start write content to content/posts/HUGO.md file and check it locally via we browser.

web is hosted by HUGO server at: http://localhost:1313

Result#

Now I have running my really simple web site with personal blog locally. Looking forward to publish it. Stay tunned ;)
